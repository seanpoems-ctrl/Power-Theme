# Thematic Scanner ‚Äî CLAUDE.md

## Project Overview

**Thematic Scanner** is a stock market analysis platform that identifies bullish thematic trends in the US equity market. It has two main scanners:

1. **Thematic Scanner** ‚Äî Daily end-of-day scrape: hierarchical theme ‚Üí sub-theme ‚Üí stock screening via Finviz
2. **Pre-Market Gapper Scanner** ‚Äî Morning pre-market gap-up analysis with AI catalyst detection (Gemini)

Deployed on **GitHub Pages** as a static React app. Data is fetched nightly via GitHub Actions, saved to `public/*.json`, and served with the built app.

## Feature Reference

### Thematic Scanner (`scraper.py`)

**Data Pipeline:**
1. Discover all themes + sub-themes from Finviz screener dropdowns
2. Score all themes with composite: `0.2√ó1W + 0.3√ó1M + 0.3√ó3M + 0.2√ó6M` ‚Üí select top 5
3. For each top theme, score + select top 5 sub-themes
4. For each sub-theme, fetch top 10 stocks with full detail
5. Enrich each stock: Finviz quote page (SMA20/50/200, ATR, ADR%) + yfinance (sparkline, earnings)
6. Build S&P 500 RS universe ‚Üí calculate RS percentile (1‚Äì99 scale) per stock
7. Fetch SPY/QQQ market indicators ‚Üí compute market signal
8. Output to `public/thematic_data.json`

**Key Functions:**
- `is_trading_day()` ‚Äî NYSE calendar check via exchange_calendars
- `discover_themes()` ‚Äî scrapes Finviz theme/sub-theme dropdowns
- `fetch_screener_stocks()` ‚Äî paginated Finviz screener (max 10 pages √ó 20 stocks)
- `fetch_stock_detail()` ‚Äî parses Finviz quote page for price, SMA ratios, ATR, ADR%, 52W high
- `fetch_sparkline()` ‚Äî 6-month daily closes via yfinance
- `fetch_earnings_yf()` ‚Äî next earnings date via yfinance
- `fetch_market_indicators()` ‚Äî SPY/QQQ SMA50/SMA200/EMA10/EMA20 metrics
- `_build_sp500_rs_universe()` ‚Äî S&P 500 list from Wikipedia + batch yfinance download
- `_composite()` ‚Äî theme/sub-theme/stock scoring formula
- `_market_signal()` ‚Äî returns `"green"` / `"yellow"` / `"red"`

**Retry logic:** 4 attempts, 180s between retries. Shorter sleeps in CI (`CI` env var detected).

### Pre-Market Gapper Scanner (`gapper_service.py`)

**Data Pipeline:**
1. Query TradingView screener: gap ‚â•5%, PM volume ‚â•200K, mkt cap ‚â•$2B, avg vol ‚â•500K, price $5‚Äì$200
2. Take top 20 gappers by pre-market % change
3. For each: fetch 24h Google News RSS headlines
4. Send to Gemini 2.5 Flash ‚Üí classify catalyst + conviction score (25‚Äì99) + trade hypothesis
5. Output to `public/gapper_data.json`

**Catalyst Categories (9):**
- Earnings, Upgrade, FDA, Thematic Narratives, Government Policy, New Contract/Partnership, Institutional Buying, Insider Buying, Others

**Conviction-based strategies:**
- Earnings/Contract ‚Üí "Gap & Go" (5-min ORB above PM high)
- Thematic/Government ‚Üí "RS Hold" (dip-buy 9-EMA/VWAP)
- Upgrade ‚Üí "Fade Candidate" (gap-fill risk)
- FDA ‚Üí "Volatility Trap" (mean reversion)

### Frontend (`src/App.js`)

**Tab 1: Thematic Scanner**
- **MarketCondition** ‚Äî Green/Yellow/Red signal with SPY/QQQ SMA metrics and trading guidance
- **Leaderboard** ‚Äî Top 5 themes ranked by perf (switchable 1D/1W/1M/3M/6M)

- **StockTable** ‚Äî Sortable, filterable table with 52W High, dist%, RVOL, avg $ vol, ADR%, RS score, vs SPY, performance cells (1D‚Äì6M), earnings date, sparkline
- **Filters** ‚Äî Min $ vol, min ADR%, min RS, max dist to 52W high

## Pure Play Detection

A stock is `pure_play: true` if it appears in **only one** sub-theme across the entire dataset. This is calculated post-collection by counting sub-theme appearances per ticker.

---

## Relative Strength (RS) Scoring

- Universe: S&P 1500 (S&P 500 + MidCap 400 + SmallCap 600, lists scraped from Wikipedia) **plus the scanned thematic stocks themselves** ‚Äî broadened so high-ADR mid-caps aren't percentile-ranked against mega-caps only
- Metric: 6-month total return
- Output: Percentile rank 1‚Äì99 (99 = top performer)
- Fallbacks: S&P 500-only if the mid/small extension fails (`_extend_rs_universe_midsmall`); internal ranking within dataset if everything fails

---

## Theme Rotation / Acceleration Rank

`_add_rotation_ranks()` annotates `theme_rankings`, `finviz_theme_rankings`, and `industry_rankings` with:

- `rank_1w` / `rank_3m` ‚Äî position when ranked by that window's perf (1 = best)
- `rotation_delta` ‚Äî `rank_3m ‚àí rank_1w`; positive = short-term rank jumped ahead of long-term rank (early rotation in)
- `accelerating` ‚Äî `true` when delta ‚â• 25% of universe size (min 3) and 1W perf > 0

Surfaced in the frontend Leaderboard as the sortable **ROT** column (‚ö° badge = accelerating).

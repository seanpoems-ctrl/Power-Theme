# Thematic Scanner — CLAUDE.md

## Project Overview

**Thematic Scanner** is a stock market analysis platform that identifies bullish thematic trends in the US equity market. It has two main scanners:

1. **Thematic Scanner** — Daily end-of-day scrape: hierarchical theme → sub-theme → stock screening via Finviz
2. **Pre-Market Gapper Scanner** — Morning pre-market gap-up analysis with AI catalyst detection (Gemini)

Deployed on **GitHub Pages** as a static React app. Data is fetched nightly via GitHub Actions, saved to `public/*.json`, and served with the built app.

## Feature Reference

### Thematic Scanner (`scraper.py`)

**Data Pipeline:**
1. Discover all themes + sub-themes from Finviz screener dropdowns
2. Score all themes with composite: `0.2×1W + 0.3×1M + 0.3×3M + 0.2×6M` → select top 5
3. For each top theme, score + select top 5 sub-themes
4. For each sub-theme, fetch top 10 stocks with full detail
5. Enrich each stock: Finviz quote page (SMA20/50/200, ATR, ADR%) + yfinance (sparkline, earnings)
6. Build S&P 500 RS universe → calculate RS percentile (1–99 scale) per stock
7. Fetch SPY/QQQ market indicators → compute market signal
8. Output to `public/thematic_data.json`

**Key Functions:**
- `is_trading_day()` — NYSE calendar check via exchange_calendars
- `discover_themes()` — scrapes Finviz theme/sub-theme dropdowns
- `fetch_screener_stocks()` — paginated Finviz screener (max 10 pages × 20 stocks)
- `fetch_stock_detail()` — parses Finviz quote page for price, SMA ratios, ATR, ADR%, 52W high
- `fetch_sparkline()` — 6-month daily closes via yfinance
- `fetch_earnings_yf()` — next earnings date via yfinance
- `fetch_market_indicators()` — SPY/QQQ SMA50/SMA200/EMA10/EMA20 metrics
- `_build_sp500_rs_universe()` — S&P 500 list from Wikipedia + batch yfinance download
- `_composite()` — theme/sub-theme/stock scoring formula
- `_market_signal()` — returns `"green"` / `"yellow"` / `"red"`

**Retry logic:** 4 attempts, 180s between retries. Shorter sleeps in CI (`CI` env var detected).

### Pre-Market Gapper Scanner (`gapper_service.py`)

**Data Pipeline:**
1. Query TradingView screener: gap ≥5%, PM volume ≥200K, mkt cap ≥$2B, avg vol ≥500K, price $5–$200
2. Take top 20 gappers by pre-market % change
3. For each: fetch 24h Google News RSS headlines
4. Send to Gemini 2.5 Flash → classify catalyst + conviction score (25–99) + trade hypothesis
5. Output to `public/gapper_data.json`

**Catalyst Categories (9):**
- Earnings, Upgrade, FDA, Thematic Narratives, Government Policy, New Contract/Partnership, Institutional Buying, Insider Buying, Others

**Conviction-based strategies:**
- Earnings/Contract → "Gap & Go" (5-min ORB above PM high)
- Thematic/Government → "RS Hold" (dip-buy 9-EMA/VWAP)
- Upgrade → "Fade Candidate" (gap-fill risk)
- FDA → "Volatility Trap" (mean reversion)

### Frontend (`src/App.js`)

**Tab 1: Thematic Scanner**
- **MarketCondition** — Green/Yellow/Red signal with SPY/QQQ SMA metrics and trading guidance
- **Leaderboard** — Top 5 themes ranked by perf (switchable 1D/1W/1M/3M/6M)
- **CorrelationGuard** — Pearson correlation warning when top 3 themes correlation >0.80
- **CounterTrendWarning** — Dead-cat bounce alert when #1 momentum theme is down >15% over 6M
- **EliteBadges** — Triple Crown, Volatility King, Liquidity Monster, VCP Tightening
- **GradeClassification** — A+ (RS≥90 + above all SMAs), A (RS≥80 + above SMA50/200), B (above SMA200)
- **TechnicalFlags** — VCP Stage 1, VDU (volume dry-up), Tight (3-day range <1.5%), ID (inside day)
- **StockTable** — Sortable, filterable table with 52W High, dist%, RVOL, avg $ vol, ADR%, RS score, vs SPY, performance cells (1D–6M), earnings date, sparkline
- **Filters** — Min $ vol, min ADR%, min RS, max dist to 52W high
- **Collapsible hierarchy** — Theme → Sub-theme → Stock table

**Tab 2: Pre-Market Gappers**
- GapperTable with catalyst badge, Gemini reasoning, trade hypothesis, conviction bar

**SVG Sparkline** — lightweight 6-month price mini-chart with gradient fill

## Development Notes

- **After changing scraper.py**: Run `python scraper.py` locally to regenerate `public/thematic_data.json`, then `npm run build` to verify the frontend renders correctly.
- **After changing gapper_service.py**: Run `python gapper_service.py` locally to regenerate `public/gapper_data.json`.
- **After changing frontend**: `npm start` for dev server, `npm run build` for production.
- **Rate limiting**: Finviz blocks aggressive scrapers. Default sleeps are 0.8–1.6s locally. Don't reduce these.
- **RS universe**: Downloads ~500 tickers from yfinance in one batch call — can be slow (~30–60s).

## Pure Play Detection

A stock is `pure_play: true` if it appears in **only one** sub-theme across the entire dataset. This is calculated post-collection by counting sub-theme appearances per ticker.

---

## Relative Strength (RS) Scoring

- Universe: All S&P 500 stocks (list scraped from Wikipedia)
- Metric: 6-month total return
- Output: Percentile rank 1–99 (99 = top performer)
- Fallback: Internal ranking within dataset if S&P 500 fetch fails

## Planned Feature: 每日選股日結報告 (Daily Sector Report)

*(Cursor plan: `每日選股日結報告_b4c7e267.plan.md`)*

A standalone script (`daily_sector_report.py`) to add after-market buy signal detection:

**Goal:** Use Gemini to identify today's top 5 sub-industries → check if Semiconductors/Technology is in the list → use yfinance to check TSM & NVDA today's gain → print "符合買入訊號" or "觀望中".

**Key design decisions:**
- Model: `gemini-1.5-flash`
- Prompt must request **pure JSON only** (no markdown wrapping), array of `{sector_name, percentage_gain}`
- Strip markdown code blocks before `json.loads()`
- Sector matching: case-insensitive keyword check for `"semiconductors"` or `"technology"`
- Buy signal condition: **both** TSM and NVDA today's gain **> 2%**
- Use `os.getenv("GEMINI_API_KEY")` with `.env` fallback
- Today's gain formula: `(close - open) / open * 100`

**Error handling:**
- Gemini failure → print message, return empty list
- `json.JSONDecodeError` → print "JSON 解析失敗", don't crash
- yfinance failure → print error, treat as 觀望中

**Note:** Gemini's sector ranking may not be real-time; actual signal is based on yfinance TSM/NVDA prices.

**Future extensions (not yet built):**
- Telegram notification on buy signal
- Save JSON to `data/daily_sector_report/YYYY-MM-DD.json`
- Integrate into existing GitHub Actions schedule

## Related Project: VCP 自動交易機器人

*(Cursor plan: `vcp_自動交易機器人_75a5e684.plan.md` — separate repo at `/Users/ryanchen/Documents/Trading_Bot_code/`)*

A fully automated VCP breakout trading bot that complements the Thematic Scanner. The scanner identifies what to watch; the bot executes trades automatically.

**Location:** `/Users/ryanchen/Documents/Trading_Bot_code/`

**Architecture (6 core modules):**

| Module | Purpose |
|--------|---------|
| 1. Market Filter | SPY/QQQ best-of-two traffic light + Top 3 Sector + Stage 2 screener → Top 5 watchlist |
| 2. VCP Recognition | Price contraction pattern detection + Pivot price calculation |
| 3. Entry Guard | Breakout validation (volume ≥1.5× 50-day avg) + fake-out filter (closes below pivot = no entry) |
| 4. Risk Management | 0.3% max loss per trade, shares = (equity×0.003) / (entry−stop), 33% max single position |
| 5. Exit Strategy | Hard stop = LOD−1%, partial profit at +20% in 3–5 days (sell 50%), trailing stop on 10-EMA |
| 6. Earnings Filter | Block entry within 14 days of earnings; force exit if <10% profit and earnings within 3 days |

**Market Signal Logic (SPY/QQQ — take the stronger):**
- 🟢 Green: price > 20MA > 50MA > 200MA, all slopes up → full buying allowed
- 🟡 Yellow: broke below 20MA or 50MA but 200MA still rising → only buy "跌市照妖鏡" stocks (above their own 20/50MA + RS at new high)
- 🔴 Red: 10MA crosses below 20MA (both falling) OR price below 200MA with flat/down slope → no new positions

**Broker:** IBKR TWS Paper Trading via `ib_insync`

**Notifications:** Telegram Bot (entry signal, exit alert, weekly report)

**Scheduler:**
- 16:30 ET daily → screening + watchlist save to `data/watchlist/YYYY-MM-DD.json`
- Every 5 min during market hours → breakout check + position management
- Friday 16:30 → weekly P&L report

**IBKR TWS Adapter status:** Completed (`ibkr_adapter.py`)
- Implements abstract `BrokerAPI` interface
- Order ID mapping: tracks `entry_vcp`, `entry_ep`, `stop`, `take_profit_A`, `eod_exit`
- Fill callbacks: `on_full_fill`, `on_partial_fill`, `on_ep_fill`, `activate_free_roll`, `handle_stop_triggered`
- Wired into `main.py` before `startup_reconciliation()`

**Stage 2 stock filter criteria:**
1. price > 50MA > 200MA, 200MA slope up for ≥1 month
2. ADR (avg daily range) ≥ 4%
3. Avg daily $ volume ≥ $10M
